Files

You have been using files for a long time. Every word processing document that you save to your computer is saved as a file. Underneath the hood there is a bit-pattern somewhere that corresponds to your file.

For text files the idea is that a file is a sequence of lines. Each line is separated by a character (or set of characters). In Unix, this is usually the newline character, frequently represented as \n. On older DOS and Windows machine the most common end-of-line sequences was \r\n (carriage return and newline.)
